ID Selectors

1- Vanilla JS ID selection uses getElementById("IDNAME") to find the div ID (NOTE: no # sign is required). This code block also makes use of the methods addEventListener() (equivalent to jQuery's click) and classList.toggle() (equivalent to jQuery's toggleClass). Notice I didn't wrap this in a function. I'm showing you that you can do JS like this but this is not best practice because it makes variable declarations global!
document.getElementById("js02ID").addEventListener("click", function() { document.getElementById("anacostia").classList.toggle("red"); });
Anacostia
2- This does the EXACT same thing as 1 using pure JS and the same methods. The difference is that I am declaring the foggyBottom function and declaring the selectors as variables. The function encapsulates this into a mini-program and properly scopes the variables to limit themselves to only this function. This is a best-practice way to write a JS function. function foggybottom ( ButtonID, SelectorID ) { var foggybottomButton = document.getElementById(ButtonID); var foggybottomSelector = document.getElementById(SelectorID); foggybottomButton.addEventListener("click", function(){ foggybottomSelector.classList.toggle("red"); }); } foggybottom( "js03ID" , "foggybottom" );
Foggy Bottom
3- This codeblock maintains the function variable and setup from 3 but uses querySelector instead of getElementById for selecting the IDs. Also note that you need to add # when you pass the ID declarations into the function (because querySelector can also find classes as well as IDs!) function mountvernon ( ButtonID, SelectorID ) { var mountvernonButton = document.querySelector(ButtonID); var mountvernonSelector = document.querySelector(SelectorID); mountvernonButton.addEventListener("click", function(){ mountvernonSelector.classList.toggle("red"); }); } mountvernon( "#js04ID" , "#mountvernon" );
Mount Vernon
